# RustDesk Server OSS

> **Status: not deployed.** Compose file is staged; this README is the playbook
> for the day you decide to bring it up. Nothing in this folder is running until
> you run step 3.

Self-hosted ID + relay for the RustDesk remote desktop client.
https://rustdesk.com/docs/en/self-host/rustdesk-server-oss/docker/

Two containers off the same image:

- **hbbs** — ID/rendezvous server. Clients register here and discover peers.
- **hbbr** — relay. Used when peers can't NAT-punch through directly.

Not Traefik-fronted — native clients use raw TCP/UDP on non-HTTP ports.

## Router port-forwards (required before clients work over WAN)

Forward all four from WAN → NAS LAN IP (`192.168.1.4`):

| Port  | Proto | Purpose                          |
|------:|:-----:|----------------------------------|
| 21115 | TCP   | hbbs — NAT test                  |
| 21116 | TCP   | hbbs — ID register / heartbeat   |
| 21116 | UDP   | hbbs — NAT hole-punch (required) |
| 21117 | TCP   | hbbr — relay traffic             |

Optional, only if you want the browser-based RustDesk web client reachable
externally: 21118/tcp (hbbs WS) and 21119/tcp (hbbr WS). Skip them otherwise.

## Bring-up steps

### 1. Create the ZFS dataset

```bash
ssh nas 'midclt call pool.dataset.create "{\"name\": \"main/appdata/rustdesk\", \"type\": \"FILESYSTEM\", \"compression\": \"LZ4\", \"atime\": \"OFF\"}"'
```

### 2. Apply the apps ACL preset

Standard 4-ACE NFSv4 ACL (apps group + postgres-in-container + owner/group).
The RustDesk image runs as root inside, so any of these would work, but using
the canonical preset keeps it consistent with every other appdata dataset.

```bash
ssh nas 'midclt call filesystem.setacl "{\"path\": \"/mnt/main/appdata/rustdesk\", \"uid\": 0, \"gid\": 568, \"acltype\": \"NFS4\", \"dacl\": [{\"tag\": \"GROUP\", \"id\": 568, \"type\": \"ALLOW\", \"perms\": {\"BASIC\": \"FULL_CONTROL\"}, \"flags\": {\"BASIC\": \"INHERIT\"}}, {\"tag\": \"USER\", \"id\": 999, \"type\": \"ALLOW\", \"perms\": {\"BASIC\": \"FULL_CONTROL\"}, \"flags\": {\"BASIC\": \"INHERIT\"}}, {\"tag\": \"owner@\", \"id\": -1, \"type\": \"ALLOW\", \"perms\": {\"BASIC\": \"FULL_CONTROL\"}, \"flags\": {\"BASIC\": \"INHERIT\"}}, {\"tag\": \"group@\", \"id\": -1, \"type\": \"ALLOW\", \"perms\": {\"BASIC\": \"FULL_CONTROL\"}, \"flags\": {\"BASIC\": \"INHERIT\"}}], \"options\": {\"recursive\": true, \"stripacl\": false}}"'
```

### 3. Register the Custom Compose app

```bash
ssh nas 'midclt -t 900 call -j app.create "{\"app_name\": \"rustdesk\", \"custom_app\": true, \"custom_compose_config\": {\"include\": [\"/mnt/main/homelab/services/rustdesk/compose.yml\"]}}"'
```

Confirm both containers are up:

```bash
ssh nas 'sudo docker ps --filter "name=hbb"'
```

### 4. Open the router port-forwards

See table at the top of this file. Without 21116/udp, NAT hole-punch fails and
every connection gets routed through the relay (slow + uses your upload bw).

### 5. Grab the server public key

The key pair is generated by hbbs on first start.

```bash
ssh nas 'cat /mnt/main/appdata/rustdesk/id_ed25519.pub'
```

### 6. Configure clients

In each RustDesk client → **Settings → Network → ID/Relay Server**:

- **ID Server:** `rustdesk.punsvik.net`
- **Relay Server:** `rustdesk.punsvik.net:21117`
- **Key:** the contents of `id_ed25519.pub` from step 5

DNS resolves `rustdesk.punsvik.net` via the wildcard A record maintained by
[ddns-updater](../ddns-updater/compose.yml) — no extra Cloudflare entry needed.

### 7. (Recommended) Lock down to known-key clients only

After step 5 confirms the key exists, append `-k _` to both commands in
`compose.yml` so the servers reject any client that doesn't present the
on-disk public key. Then redeploy:

```bash
ssh nas 'midclt -t 300 call -j app.redeploy "\"rustdesk\""'
```

## Update later

```bash
# Optionally pin RUSTDESK_IMAGE_TAG in .env first
ssh nas 'midclt -t 600 call -j app.update "\"rustdesk\""'
```

## Storage layout

- `/mnt/main/appdata/rustdesk/` — shared by both containers
  - `id_ed25519` / `id_ed25519.pub` — server key pair (**back these up** — losing
    them means every client has to be reconfigured with the new key)
  - `db_v2.sqlite3` — registered clients / peer table

## Things to know before re-reading this in a year

- **OSS edition has no admin UI.** All configuration is the CLI flags in
  `compose.yml` plus the files in the data directory. Pro edition has a web UI
  but is paid + per-device licensed.
- **Relay bandwidth** flows through the home upload when peers can't
  direct-connect. Fine for 1–2 sessions, painful on a slow link.
- **`-r` flag in `hbbs` command** hardcodes the relay endpoint clients are told
  to use. If `rustdesk.punsvik.net` ever moves, change it there.
- **CGNAT check:** if your ISP starts using CGNAT, port forwards stop working
  and you'd need a tunnel — Cloudflare Tunnel doesn't help (it's HTTP-only).
- **Homepage tile** is not registered. Add an entry to
  [services/homepage/services.yaml](../homepage/services.yaml) under Backend
  when you bring this up — track the `hbbs` container for status, with `href`
  pointing to rustdesk.com since there's no local web UI to link to.

## Decommission

```bash
ssh nas 'midclt call app.delete "\"rustdesk\""'
ssh nas 'midclt call pool.dataset.delete "[\"main/appdata/rustdesk\", {\"recursive\": false, \"force\": false}]"'
```

Don't forget to close the router port-forwards.
